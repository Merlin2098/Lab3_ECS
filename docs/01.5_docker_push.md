# 01.5 — Build y push de imagen Docker a ECR

Construye la imagen de la aplicación FastAPI y la sube al repositorio ECR.
Ejecutar **después** de `01_backend_bootstrap.md` y **después** de
`terraform apply` en `infra/` (Paso 1 de `02_infra_deploy.md`), que crea el repositorio ECR.

> El repositorio ECR debe existir antes de poder hacer `docker push`.

---

## Windows (PowerShell)

### Paso 0 — Cargar credenciales AWS

```powershell
Get-Content infra\env\.env.credentials | ForEach-Object {
  if ($_ -match "^\s*#" -or $_ -match "^\s*$") { return }

  $name, $value = $_ -split "=", 2
  [Environment]::SetEnvironmentVariable($name.Trim(), $value.Trim(), "Process")
}

# Verificar
aws sts get-caller-identity
```

### Paso 1 — Build y push

```powershell
# Obtener Account ID y URL del repositorio desde terraform output
$ACCOUNT_ID = aws sts get-caller-identity --query Account --output text
$REGION     = "us-east-1"

Push-Location infra
$REPO_URL = terraform output -raw repository_url
Pop-Location

Write-Host "Repositorio ECR: $REPO_URL"

# Login en ECR
aws ecr get-login-password --region $REGION |
  docker login --username AWS --password-stdin "${ACCOUNT_ID}.dkr.ecr.${REGION}.amazonaws.com"

# Build de la imagen desde app/
docker build -t ecs-terraform-lab app/

# Tag con la URL del repositorio
docker tag ecs-terraform-lab:latest "${REPO_URL}:latest"

# Push a ECR
docker push "${REPO_URL}:latest"

Write-Host "Imagen publicada en: ${REPO_URL}:latest"
```

## Linux / macOS (bash)

### Paso 0 — Cargar credenciales AWS

```bash
set -o allexport
source <(grep -v '^\s*#' infra/env/.env.credentials | grep -v '^\s*$')
set +o allexport

# Verificar
aws sts get-caller-identity
```

### Paso 1 — Build y push

```bash
# Obtener Account ID y URL del repositorio desde terraform output
ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
REGION="us-east-1"

pushd infra
REPO_URL=$(terraform output -raw repository_url)
popd

echo "Repositorio ECR: $REPO_URL"

# Login en ECR
aws ecr get-login-password --region "$REGION" | \
  docker login --username AWS --password-stdin \
  "${ACCOUNT_ID}.dkr.ecr.${REGION}.amazonaws.com"

# Build de la imagen desde app/
docker build -t ecs-terraform-lab app/

# Tag con la URL del repositorio
docker tag ecs-terraform-lab:latest "${REPO_URL}:latest"

# Push a ECR
docker push "${REPO_URL}:latest"

echo "Imagen publicada en: ${REPO_URL}:latest"
```

```
latest: digest: sha256:... size: ...
Imagen publicada en: <account_id>.dkr.ecr.us-east-1.amazonaws.com/ecs-terraform-lab-dev:latest
```

## Re-deploy (actualizar imagen existente)

Si modificas la app y quieres forzar que ECS descargue la nueva imagen:

```powershell
# Recargar credenciales si la sesión cambió
Get-Content infra\env\.env.credentials | ForEach-Object {
  if ($_ -match "^\s*#" -or $_ -match "^\s*$") { return }
  $name, $value = $_ -split "=", 2
  [Environment]::SetEnvironmentVariable($name.Trim(), $value.Trim(), "Process")
}

# Obtener URL del repo
Push-Location infra
$REPO_URL = terraform output -raw repository_url
Pop-Location

$ACCOUNT_ID = aws sts get-caller-identity --query Account --output text
$REGION     = "us-east-1"

# Login, build, tag y push
aws ecr get-login-password --region $REGION |
  docker login --username AWS --password-stdin "${ACCOUNT_ID}.dkr.ecr.${REGION}.amazonaws.com"

docker build -t ecs-terraform-lab app/
docker tag ecs-terraform-lab:latest "${REPO_URL}:latest"
docker push "${REPO_URL}:latest"

# Forzar nueva task en ECS
Push-Location infra
aws ecs update-service `
  --cluster (terraform output -raw cluster_name) `
  --service (terraform output -raw service_name) `
  --force-new-deployment
Pop-Location
```

set -o allexport
source <(grep -v '^\s*#' infra/env/.env.credentials | grep -v '^\s*$')
